# AptiForge

Personalized project recommender for developers.

AptiForge helps you discover coding project ideas from your **resume**, **GitHub activity**, and **learning goals**. It extracts skills, ranks a project catalog with skill overlap + TF-IDF goal similarity, and exports a roadmap.

## Features

- Upload a resume (**PDF**, **DOCX**, or **TXT**) — or **try the sample resume**
- Analyze a public **GitHub** profile for languages and strengths
- Enter **learning goals**, choose **comfort/stretch** fit, and filter by **track**
- Get personalized suggestions with tech stack, difficulty, time estimate, skill gaps, and **why it fits**
- Interactive **milestones**, **compare** up to 3 projects, and coaching (**interview tips / README / weekly plan**)
- **Save/export** roadmap as Markdown, JSON, or PDF; **share** via URL; browse **recent runs**
- Dark mode, forge reveal animations, and lightweight **usage** stats

## Deploy on Vercel

AptiForge is a Flask app. Vercel needs an `api/` entrypoint and root `requirements.txt`.

1. Push to GitHub
2. Import the repo at [vercel.com/new](https://vercel.com/new)
3. Leave build settings as detected (Python via `api/index.py`)
4. Redeploy

Optional env vars in Vercel Project Settings:
- `ANTHROPIC_API_KEY` — richer “why it fits” + custom project generation
- `ANTHROPIC_MODEL` — default `claude-sonnet-4-5`
- `PINECONE_API_KEY` + `PINECONE_INDEX` — optional vector search (create index with **dimension 64**, metric **cosine**)
- `GITHUB_TOKEN` — higher GitHub API limits

Local Flask still works with `cd backend && python app.py`.

## AI features

- **Ranking** stays skill + semantic (local TF-IDF, or Pinecone when configured)
- **Anthropic Claude** (optional) rewrites personalized “why it fits” explanations
- **Custom project** is generated by Claude when catalog fit is weak
- Toggle **Use Anthropic…** in the UI, or send `use_ai: false` in the API

## Setup

```bash
cd backend
python -m venv ../.venv

# Windows
..\.venv\Scripts\activate

# macOS / Linux
# source ../.venv/bin/activate

pip install -r requirements.txt
copy ..\.env.example ..\.env   # optional: set GITHUB_TOKEN
```

## Run

```bash
cd backend
python app.py
```

Open **http://127.0.0.1:5000/** for the UI.

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/` | Web UI |
| GET | `/api` | API info |
| GET | `/sample_resume` | Bundled demo resume |
| POST | `/upload_resume` | Extract skills from a resume file |
| POST | `/analyze_resume` | Skills + optional GitHub/goals + recommendations |
| POST | `/analyze_github` | JSON `{"username": "..."}` |
| POST | `/recommend_projects` | JSON skills + learning goals (+ fit_mode/tracks) |
| POST | `/export_roadmap` | JSON projects → roadmap Markdown/JSON |
| POST | `/project_assist` | Interview tips / README / weekly plan |
| GET/POST | `/analytics` | Lightweight usage counters |

### Optional Streamlit UI

Requires `streamlit` (may need a separate install if wheels are unavailable on your platform):


## Tech stack

- **Backend:** Flask, Flask-CORS, requests, pypdf, python-docx
- **Ranking:** skill intersection + scikit-learn TF-IDF cosine similarity
- **Frontend:** HTML/CSS/JS served by Flask (Streamlit optional)
- **Optional env:** `GITHUB_TOKEN` (see `.env.example`)
